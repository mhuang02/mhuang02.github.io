Analysis of League of Legends Games (change title later)

Matthew Huang, Isaac Chen, Ingrette Wang

Released in October 2009, League of Legends has solidified itself as one of the most popular played video games for the past decade. At its core, League of Legends is a 5v5 MOBA (multiplayer online battle arena) game where two teams of players face off against one another, and race to see who can destroy the other team’s base first. Although the game has evolved over its lifespan, the core gameplay of teams trying to create an advantage over the team has stayed consistent. Being a complex game with many factors influencing a game’s state and outcome, are there any specific factors that can predict the outcome of a game? These are some questions that we aim to answer in this tutorial.

What features make up a game of League of Legends?

The game takes place on a map with a main base (nexus), with three main lanes expanding out from the main base. Each lane contains an inhibitor, and three towers that protect enemies from entering the main base. The main objective of the game is to destroy the enemies main base by marching down one(or multiple) lanes, destroying towers and inhibitors along the way. Throughout the game, players from both teams are able to unlock and upgrade their champions’ abilities as they level up, which can be achieved through a couple of different ways. Players can level up by defeating enemy champions in combat, granting them experience, as well as gold, which can be used to buy items in an item shop that increases a player’s stats (attack damage, magic damage, movement speed, health, to name a few), which creates an advantage for that player. Another way that players can level up is through killing lane minions, which are NPC’s (non-playable characters) that spawn every 30 seconds and grant gold and experience upon death.

More in-depth information about the game can be found here
https://en.wikipedia.org/wiki/League_of_Legends

In this tutorial, we will be analyzing how multiple factors such as gold difference, team color, (add more) from around 10,000 Diamond ranked games, can predict a game’s outcome. By analyzing this data, we hope to better explore the different facets that contribute toward game results, and gain insights into strategies and patterns that lead to success in the competitive scene.

Data Curration

Our data was obtained through the Riot Games API, and was obtained through an uploader who classified the data as public domain. The format of the original data is contained in a csv file that contains 9,879 entries.
Because the raw data is already in numerical format so there is no need to convert our data to integers. However, we will be deleting some uncessary columns such as (blueFirstBlood, redFirstBlood, gameId, blueEliteMonsters, and redEliteMonsters) Based off our own experiences that we have from playing the game, these data values don't provide much value when it comes to analysis of game outcomes. The advantage earned from getting first blood is only 150 gold, which is pretty negligible in the grand scheme of the game. The number of elite monsters killed is also minor, as only one elite monster spawns within the first ten minutes of the game, and the buff it provides isn't enough to determine a game's outcome.

Now that our data is more cleaned up, we can use visualization to better see our data and how it relates to each other. To do this, we want to see whether any of these columns correlate with one another.

As League of Legends is broadly a resource gathering game (of the in-game currency, gold), gaining more gold generally leads to resource advantages that could allow the team with more gold to win the game. Total minions killed, kill count, CS per minute, objectives killed, assists, and first kill counts also influence the gold per minute, as killing minions, champions, and objectives all grant the player gold in the game. Total minions killed and objectives killed also influences CS (creep score) per minute, as minions and objectives killed contribute to a player’s overall creep score.

This means that there should be features that correlate with one another!

            > jupyter nbconvert --to latex mynotebook.ipynb

            Both HTML and LaTeX support multiple output templates. LaTeX includes
            'base', 'article' and 'report'.  HTML includes 'basic', 'lab' and
            'classic'. You can specify the flavor of the format used.

            > jupyter nbconvert --to html --template lab mynotebook.ipynb

            You can also pipe the output to stdout, rather than a file

            > jupyter nbconvert mynotebook.ipynb --stdout

            PDF is generated via latex

            > jupyter nbconvert mynotebook.ipynb --to pdf

            You can get (and serve) a Reveal.js-powered slideshow

            > jupyter nbconvert myslides.ipynb --to slides --post serve

            Multiple notebooks can be given at the command line in a couple of
            different ways:

            > jupyter nbconvert notebook*.ipynb
            > jupyter nbconvert notebook1.ipynb notebook2.ipynb

            or you can specify the notebooks list in a config file, containing::

                c.NbConvertApp.notebooks = ["my_notebook.ipynb"]

            > jupyter nbconvert --config mycfg.py

To see all available configurables, use `--help-all`.
